# ⚽ Football Free Kick Analyzer

A production-ready FastAPI server that analyzes football free-kick videos using computer vision. It detects kick events, splits the video into individual clips, and annotates each clip with player pose, ball bounding box, and ball trajectory.

## How It Works

```
Upload Video
     │
     ▼
┌─────────────────┐
│  Video Loader   │  Validates format, reads metadata (fps, dimensions, etc.)
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  Kick Detector  │  Samples frames at ~15fps, runs YOLO ball detection on each.
│                 │  Detects kicks via:
│                 │    A) Ball disappearance (foot contact)
│                 │    B) Sudden ball displacement (rapid acceleration)
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ Clip Extractor  │  For each kick: extract window (1.5s before → 2.5s after)
│                 │  and for EVERY frame in that window run:
│                 │    • YOLO  → ball bounding box
│                 │    • MediaPipe → player keypoints + skeleton
│                 │    • Annotator → draws overlays, trajectory trail, HUD
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  JSON Response  │  Returns kick count, timestamps, clip paths, detection stats
└─────────────────┘
```

## Prerequisites

- **Python 3.10+**
- **pip**
- No GPU required — everything runs on CPU

> **Note on model downloads:** On the first API call, the server will automatically download:
> - `yolov8n.pt` (~6 MB) — YOLO ball detection model
> - MediaPipe pose model (~30 MB)
>
> These are cached locally and not re-downloaded on subsequent runs.

## Installation

### 1. Clone / download the project

```bash

```

### 2. Create a virtual environment (strongly recommended)

```bash

# Activate it:
```



## API Reference

### `POST /api/v1/analyze`

Upload a free-kick video for analysis.

**Request**
- Method: `POST`
- Content-Type: `multipart/form-data`
- Field: `video` — the video file

**cURL example**
```bash
curl -X POST http://localhost:8000/api/v1/analyze \
  -F "video=@/path/to/freekick.mp4"
```

**Error Responses**

| Status | `error_code`         | Cause                              |
|--------|---------------------|------------------------------------|
| 400    | `UNSUPPORTED_FORMAT` | File is not a recognised video type |
| 400    | `VIDEO_LOAD_ERROR`   | File is corrupted or unreadable     |

| 422    | `NO_KICKS_DETECTED`  | No kick events found in the video   |

## Configuration

All settings live in `.env` (copy from `.env.example`). Key options:

| Variable | Default | Description |
|---|---|---|
| `PROCESSING_FPS` | `15` | Frames/sec scanned during kick detection. Lower = faster but less accurate. |
| `YOLO_MODEL_SIZE` | `yolov8n.pt` | YOLO model. `n`=fastest, `s`/`m`=more accurate (but slower on CPU). |
| `BALL_CONFIDENCE_THRESHOLD` | `0.3` | Min YOLO confidence to accept a ball detection. |
| `MIN_KICK_DISPLACEMENT` | `40.0` | Min pixel jump to count as a kick signal. |
| `CLIP_PRE_KICK_SECONDS` | `1.5` | Seconds before kick to include in clip. |
| `CLIP_POST_KICK_SECONDS` | `2.5` | Seconds after kick to include in clip. |
| `MAX_VIDEO_SIZE_MB` | `500` | Upload size limit. |

## Troubleshooting

**"No kicks detected"**
- The ball may be hard for YOLO to detect in your video quality. Try lowering `BALL_CONFIDENCE_THRESHOLD` to `0.2`.
- Also try lowering `MIN_KICK_DISPLACEMENT` to `25.0`.

**Processing is very slow**
- On CPU a 2-minute video typically takes 2–5 minutes. This is expected.
- Switch `YOLO_MODEL_SIZE` to `yolov8n.pt` (nano) if you're using a larger model.
- Lower `PROCESSING_FPS` to `10` to sample fewer frames during kick detection.

**MediaPipe not detecting the player**
- Lower `POSE_MIN_DETECTION_CONFIDENCE` to `0.4`.
- Ensure the player is reasonably in frame and well lit.

**YOLO model not downloading**
- Check your internet connection on first run. YOLO downloads `yolov8n.pt` automatically from Ultralytics servers.

**Port already in use**
- Change `APP_PORT` in `.env` to another port (e.g. `8001`).
